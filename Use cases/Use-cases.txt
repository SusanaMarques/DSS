Use case: Upload de musica e videos
Ator: User registado / Admin
Pré Condição: Ator está logged in
Pós Condição: A musica foi carregada
Fluxo normal:
    1- Ator insere as musicas
    2- Sistema verifica duplicaçoes e formato
    3- Sistema coloca as musicas nas suas bibliotecas 
    4- Sistema classifica as musicas
    5- Verifica users com conteúdo em comum
    6- Adiciona users com conteúdo em comum á lista de potenciais amigos
    7- Mensagem de Sucesso
Fluxo Alternativo 1:
    2.1- Sistema deteta duplicaçoes
    2.2- Sistema envia mensagem de aviso
    2.3- (5)
Fluxo Alternativo 2:
    5.1- Não tem conteúdo em comum com outros utilizadores
    5.2- (7)
Fluxo de exceção:
    2.1- Sistema deteta formato Errado
    2.2- Mensagem de Erro

Use case: Login
Ator: User registado/ convidado / Admin
Pré Condição: Sistema está livre
Pós Condição: Utilizador autenticado e conectado ao sistema
Fluxo normal:
    1- Sistema apresenta o tipo de Login
    2- User escolhe o registado e introduz credenciais
    3- Sistema valida credenciais
    4- Sistema apresenta as bibliotecas
Fluxo Alternativo 1:
    2.1- User escolhe convidado
    2.2- (4)
Fluxo de exceção:
    2.1- Sistema não valida as credenciais
    2.2- Mensagem de erro
